# V1 to V2 guide

Source: https://finsweet.com/client-first/docs/v1-to-v2

## Why V2 exists

The most significant update to V2 is **how Client-First concepts are
explained**. V1 lacked organization of content. V2 introduces a clearer
content organization method.

Each topic has a more precise explanation under the "strategy" naming:

- Classes strategy
- Core Structure strategy
- Typography strategy
- Spacing strategy
- And more

## What's new in V2

### Folders

Folders redefines how we visualize classes. Folders is a virtual folder
system in Finsweet Extension.

- Underscore in class name → folder.
- Dashes only in class name → utility class in Utility folder.
- See `references/08-folders.md` and `references/09-folders-strategy.md`.

### Components are better defined

- **V1:** any underscore = component.
- **V2:** underscore = folder. Use the word `component` as the element
  identifier to mark a class as a complete reusable component.

```
clients-slider_component   # the parent wrapper of a complete component
clients-slider_mask
clients-slider_grid
clients-slider_arrow
```

### Class types are better defined

Custom class, utility class, global class, combo class are now clearly
distinguished. See `references/01-classes-strategy-1.md`.

### `page-padding` is now `padding-global`

The most important padding class is renamed for better folder
organization. Instead of living in the `page-` folder, it lives in the
`padding-` folder.

### `section-[identifier]` is now `section_[identifier]`

Section classes now use underscore. Utility classes use only dashes. So
sections — which are custom classes — use underscore, which puts all
sections in the `section_` folder.

### New `padding-section-[size]`

New utility class system to manage global vertical spacing across sections.

```
padding-section-small
padding-section-medium
padding-section-large
```

### New `heading-style-[h#]`

Change a Heading's style without changing its tag.

```
H1 + heading-style-h2   # visually H2, semantically H1
```

### New `display-inlineflex`

`display-inlineflex` applies `display: inline-flex`, which is not supported
natively in Webflow. Useful for buttons that need flex behavior but should
not take 100% of the space.

### Removed `show-*` classes

`show-*` classes (display: block at responsive levels) are removed. They
were too limiting and led to responsive conflicts.

Use a custom class or combo class instead:

```
modal_component
modal_component is-visible-tablet
```

### New spacing strategies

- **Utility spacing:** spacing wrappers (V1) and spacing blocks (V2).
- **Custom spacing:** on the element (V1) and CSS Grid on the parent (V2).

### Improved custom CSS in Global Styles embed

See `references/15-global-styles-embed.md` for the full list of CSS
snippets and what they do.

## Migration cheat sheet (v1 → v2)

| V1 | V2 |
| --- | --- |
| `page-padding` | `padding-global` |
| `section-home` | `section_home` |
| `show-tablet` | Custom class with `display: block` on tablet |
| Heading tag mismatch | `heading-style-h#` |
| No section padding | `padding-section-[size]` |
| Underscore = component | `[folder]_component` (explicit keyword) |

## What to do if you have a V1 project

1. Read the V2 docs (this skill covers all of them).
2. Identify the V1 → V2 renames in your project.
3. Rename `page-padding` → `padding-global` and `section-*` → `section_*`.
4. Use Folders by Finsweet Extension to visualize the new structure.
5. Add `padding-section-*` to your existing sections where appropriate.
6. Use `heading-style-h#` to keep Heading tags and styles in sync.
7. Replace any `show-*` classes with custom class or combo behavior.
8. Adopt the spacing block strategy (`spacer-*`) for new spacing.

There is no automated migration tool. The renames are a find-and-replace
exercise, and Folders (Finsweet Extension) can bulk rename folders.
